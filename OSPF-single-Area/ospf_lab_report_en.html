1. Project Description and Topology Overview

The objective of this practical lab is to construct and secure an integrated dynamic routing environment utilizing OSPF Area 0 across four core internal routers (R1, R2, R3, and R4). Additionally, it simulates external internet connectivity by linking the main edge router, R1, to an Internet Service Provider router (ISPR1) via a public IP subnet range.

In this scenario, the role of an Autonomous System Boundary Router (ASBR) is implemented on R1. A static default route pointing toward the ISP gateway next-hop is established, which is subsequently injected and dynamically propagated to all downstream routers within the internal OSPF domain. This setup guarantees that all local area network nodes automatically acquire a path to external networks without relying on manual configuration.

2. Configuration Commands Applied to Edge Router R1

The interface facing the ISP is configured, an active static default route is established to avoid recursive lookup issues, and the OSPF process is initialized to inject and distribute the default path to the rest of the enterprise topology:

R1(config)# interface GigabitEthernet3/0 R1(config-if)# ip address 203.0.113.1 255.255.255.252 R1(config-if)# no shutdown R1(config-if)# exit R1(config)# router ospf 1 R1(config-router)# router-id 1.1.1.1 R1(config-router)# network 1.1.1.1 0.0.0.0 area 0 R1(config-router)# network 10.0.12.0 0.0.0.3 area 0 R1(config-router)# network 10.0.13.0 0.0.0.3 area 0 R1(config-router)# passive-interface Loopback0 R1(config-router)# default-information originate R1(config-router)# exit R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2

4. Engineering Conclusions & Lessons Learned

The infrastructure converges flawlessly following the deployment of appropriate interface matching parameters. Primary engineering insights derived from this implementation include:

  • Resolution of Recursive Lookup Faults: A static default route next-hop must reside within an explicitly reachable, directly connected subnet structure. Failing this, the routing engine drops the path candidacy, halting propagation entirely.
  • Scalable Dynamic injection: Utilizing default-information originate abstracts administrative overhead, preventing manual misconfigurations across scaling core frameworks.
  • Symmetric Routing Redundancy: The lower boundaries showcase proper execution of Equal-Cost Multi-Pathing (ECMP), providing optimized load redistribution and deterministic fault-recovery windows.
